# Decap CMS Setup for CorpusAbierto (Hugo Site)

## Files
- Place the `admin/` folder inside your Hugo site's `static/` directory.
- Hugo will serve `/static/admin/` as `/admin/` on your website.

## GitHub OAuth Setup (Required for authentication)
1. Visit: https://github.com/settings/developers
2. Click "New OAuth App"
3. Fill in:
   - **Application name**: CorpusAbierto CMS
   - **Homepage URL**: https://corpusabierto.com
   - **Authorization callback URL**: https://corpusabierto.com/admin/

4. After creation, note your **Client ID** and **Client Secret**.

5. Use the **Decap Community Identity Gateway**:
   In `config.yml`:
   ```yaml
   base_url: https://identity.decapcms.org
   auth_endpoint: auth
   ```

## Security Notes
- Users must authenticate via GitHub to edit content.
- Only repo collaborators can commit changes.
- Your config is safe to store publicly (it contains no secrets).

## Editing Partial Sections
The `config.yml` exposes only relevant frontmatter + content fields like:
- Título
- Obra
- Libro
- Autor
- Texto original (Markdown)
- Traducción (Markdown)
- Notas críticas / del traductor

You can add more fields or split collections as needed.

## Deployment
1. Commit and push `static/admin/` to your GitHub Pages repo.
2. Open `https://corpusabierto.com/admin/`
3. Log in via GitHub.
